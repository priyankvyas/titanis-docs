  •  package titanis

    Titanis is a machine learning and data analysis platform tailored to optimize traditional machine learning algorithms for streaming input data, leveraging a distributed system architecture.

    Titanis is a machine learning and data analysis platform tailored to optimize traditional machine learning algorithms for streaming input data, leveraging a distributed system architecture. runTask is the primary entry point of the application. Titanis has native implementations of kNN and Hoeffding Tree. It can also utilize implementations of MOA learners from the MOA library.

    More native learner and helper implementations are to be added in future versions.

Type Members

  1.  class CUSUM extends DriftDetector

    The drift detector that uses the CUSUM drift detection method to detect concept drift.

    The drift detector that uses the CUSUM drift detection method to detect concept drift.

    The false alarm rate can be configured using the false alarm rate option.

    The drift detector has global variables to retain the incremental sums from the previous batch. The accumulators are used to maintain the incremental sums while processing the batch. For each of the required incremental sums, a User Defined Function is used to update the accumulator and correct the sum with the previous batch's results when required. The initOverride method initializes all the required variables and accumulators. The detect method initiates the detection streaming query that adds the necessary columns and outputs the stream with the detectedChange column.

  2.  trait DriftDetector extends Configurable with HasLabelCol with HasIDCol

    Defines a Drift Detector.

    Defines a Drift Detector.

    A drift detector is configurable. The threshold for detecting change and the minimum number of instances before change can be detected can be configured using the corresponding options. The schema of the input data stream and the threshold are stored as global variables. The isReset flag is used to indicate if concept drift has been detected in this batch.

    A Drift Detector has a Monitor that is initialized in the init method. The Monitor records the processing duration of the streaming queries and writes the resulting DataFrames to a CSV file.

    A Drift Detector overrides the copy method inherited from the Params traits. It returns a default copy of the Drift Detector object with extra parameters.

    Used by CUSUM, PageHinkley, and MOADriftDetector.

    Since

    0.0.1

  3.  class MOADriftDetector extends DriftDetector with HasMOADetectorObject

    The drift detector that uses the MOA drift detection methods to detect concept drift.

    The drift detector that uses the MOA drift detection methods to detect concept drift.

    The drift detector from MOA can be chosen using the moaDriftDetector option. The drift detector is retained as a global object.

    The getChange User Defined Function is used to train the drift detector from MOA on the instances from the input stream. The initOverride method initializes the chosen MOA drift detector. The detect method initiates the detection streaming query that adds the necessary columns and outputs the stream with the detectedChange column.

    Since

    0.0.1
