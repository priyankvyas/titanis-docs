  •  package titanis

    Titanis is a machine learning and data analysis platform tailored to optimize traditional machine learning algorithms for streaming input data, leveraging a distributed system architecture.

    Titanis is a machine learning and data analysis platform tailored to optimize traditional machine learning algorithms for streaming input data, leveraging a distributed system architecture. runTask is the primary entry point of the application. Titanis has native implementations of kNN and Hoeffding Tree. It can also utilize implementations of MOA learners from the MOA library.

    More native learner and helper implementations are to be added in future versions.

  1.  class EvaluateConceptDrift extends Task

    Task used to detect concept drift on a data stream.

    Task used to detect concept drift on a data stream.

    The task evaluates the concept drift in the values of an attribute of the input data. The Drift Detectors available in this version are CUSUM and Page-Hinkley. Future versions will have more options for drift-detection algorithms. Drift detectors from MOA can also be used using the MOADriftDetector option. The individual drift detectors are configurable.

    This task executes one query, the Detect query, for each batch of input data received.

    Since

    0.0.1

  2.  class EvaluateLearner extends Task

    Task used to evaluate a learner on a data stream.

    Task used to evaluate a learner on a data stream.

    The task evaluates the performance of a Learner in a learning task on the input data. The Learners available in this version are kNN and OzaBag. Future versions will have more options for learning algorithms. The individual learners are configurable.

    This task follows a predict-evaluate-train cycle for each batch of input data received.

    Since

    0.0.1

  3.  trait Task extends Configurable

    Defines a Task.

    Defines a Task.

    A Task in Titanis must contain an execute method that performs a specific task. The Task is usually evaluative in nature, where the performance of the learner or an analyzer is measured during an experiment. Tasks may differ in the configurable options they provide or the logic of the execute method.

    The task uses a Stream Reader to read in the input data. The only Stream Reader available in this version is the Directory Reader. Future versions will have more options to read from different sources. The Directory Reader is configurable.

    If the target attribute and the weight attribute are given a different name in the schema of the input data, it has to be mentioned in the configuration using the -t and -w options respectively, to avoid any runtime errors.

    The learning task is determined based on the type of the target attribute in the schema (classification for nominal and regression for numeric) and the corresponding evaluators should be used to avoid any runtime errors (Basic Classification Evaluator for classification and Basic Regression Evaluator for regression). The evaluators are configurable.

    Used by EvaluateLearner and EvaluateConceptDrift.

    Since

    0.0.1
